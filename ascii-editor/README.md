# Vignette Bloom — Canvas2D

This folder contains a from-scratch recreation of the animated **Vignette Bloom** raster effect. It does not import the original 21st.dev implementation.

Open `index.html` through any static web server to compare the supplied `ref-008.webp` source with the DoctorGuidance profile portrait. The demo exposes the full list of render modes and a focused set of live controls.

## Pipeline

1. Draw and crop the source photo, then create the selected background layer.
2. Average the color and luminance of every grid cell and calculate local edge energy.
3. Render the selected primitive with coverage, density, tone-curve, inversion, and animation modulation.
4. Apply brightness, contrast, saturation, grayscale, tint/blend, and optical blur in that order.
5. Layer enabled post-effects, point lights, and finally the plain-photo reveal mask.

Implemented primitives: characters, dither, mosaic, pixel, dots, cross, diamond, voxel, lego, mixed, lines, diagonal, braille, disco, hexdump, matrix, rings, hearts, stars, hexagons, triangles, bubbles, hatch, contour, and halfblocks.

Implemented post-effects: scan lines, vignette, bloom, chromatic split, film grain, glitch slices, halftone, pixelation, and film dust. Gaussian, directional, tilt-shift, lens, and progressive blur modes share the same ordered pipeline.

## Re-render the GitHub portrait

GitHub profile READMEs do not execute JavaScript or Canvas. The README therefore embeds a pre-rendered animated WebP generated by the same Canvas2D engine.

```bash
npm install
npm run render:profile
```

The command reads `demos/generated/profile-source.png` and writes `../assets/profile-vignette-bloom.webp` without relying on GitHub Actions.

Run `npm run validate:canvas` to exercise every primitive, blur path, post-effect, point-light layer, and reveal-mask path against a small raster canvas.
